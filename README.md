# 心得总结

## 项目结构总结

### 命名规则 
1. 文件的名称出现多个单词，需要使用下划线 “_”隔开
2. 文件名和包名一般直接将单词进行拼接，如果文件名和 包名不同，需要在导入的时候，在导入内容前面加上包名
3. 模块名 使用 连字符 “-” 分隔多个单词


### 单元测试
1. 文件名必须是 "_test" 结尾
2. 包名和待测试的go文件同名同目录，这样才能调用私有方法


### 其余测试
1. 其他测试，例如端到端测试或者集成测试，请额外创建一个 test 文件夹
